# Летнее бинго — общее ТЗ (master)

> Корневой документ-источник. На его основе собраны два дочерних ТЗ:
> **(А)** `letnee-bingo-design-tz.md` — для Claude Design (визуальный язык, марки, состояния);
> **(Б)** `letnee-bingo-code-tz.md` — для Claude Code (сборка статического сайта).
> Разделы 1–7 — общий контекст. Этапы 1–2 → дизайн-ТЗ, этапы 3–5 → код-ТЗ.

---

## 1. Контекст проекта

**Что это.** Минималистичный интерактивный сайт — дополнение к онлайн-курсу по фотографии (3 недели, ~20 участниц).

**Зачем.** Практика замечания и присутствия. В течение лета участница «собирает» приятные летние моменты, отмечая выполненные наблюдения. Сайт добавляет ценности курсу и даёт тёплые тактильные впечатления.

**Главный принцип (сознательный парадокс).** Это курс по фотографии, но практика намеренно **без камеры**. Идея — соединяться с моментами лета, не беря в руки телефон/фотоаппарат. **Загрузки фотографий нет** — это не альбом снимков, а альбом замеченного.

**Тон.** Тихий, созерцательный, тактильный. Не игра-соревнование, а личный ритуал-альбом.

---

## 2. Суть (core concept)

- Поле в виде листа бумаги с сеткой **4×5 = 20** ячеек-наблюдений.
- Каждое задание — короткая сенсорная фраза («намочить ноги в воде»).
- Тап по заданию = «замечено». Текст плавно исчезает и превращается в маленькую **почтовую марку** (иллюстрацию).
- Участница не вычёркивает, а **собирает коллекцию марок** — личный альбом лета, наполняющийся постепенно.
- Когда собраны все 20 — мягкий финал: фраза **«Лето замечено»**. Это не «победа», а тихая открытка, которая долетела до человека и поощряет его за внимание, которое он даёт своей жизни.

---

## 3. Что НЕ входит (границы / non-goals)

- ❌ Загрузка или привязка фотографий участниц. Сознательно — практика без камеры.
- ❌ Учёт южного полушария / сезонная адаптация. Делаем «северное лето».
- ❌ Аккаунты, логин, бэкенд. Всё локально в браузере.
- ❌ Социальный/общий слой (общие стены, ленты, сравнение участниц). Опыт строго индивидуальный.
- ❌ Геймификация с давлением: ни таймеров, ни «осталось N до победы», ни стриков.

## 4. Контент (копирайт — сердце проекта)

Главная ценность — **в фразах**, а не в механике. К копирайту относимся как к основному дизайн-материалу.

**Требования к фразам:**
- 20 фраз (= размер сетки).
- Регистр: **её голос, её способ видеть**. Сенсорно, конкретно, тихо. НЕ дженерик из wellness-приложений.
- Форма: короткое наблюдение/побуждение, ~3–6 слов, глагол в инфинитиве. Одно ощущение на карточку.
- Тематические оси для баланса: запахи · прикосновения · свет · звук · вкус · движение · вода · небо/облака · тепло.
- **Финальную редакцию пишет она.** Ниже — стартовый материал.

**Из брифа (опорные):**
1. Понюхать запах свежескошенной травы
2. Найти животное в облаках
3. Намочить ноги в воде
4. Посмотреть закат
5. Растереть в ладонях травы и вдохнуть их аромат
6. Понаблюдать за отражениями

**Черновик-иллюстрация регистра (финал — за ней):**
- Поймать ладонью тёплый ветер
- Не прятаться от тёплого дождя
- Пройтись босиком по нагретой земле
- Заметить, как свет меняется к вечеру
- Найти самый тихий звук вокруг
- Дать солнцу нагреть закрытые глаза

**Финальная строка:** «Лето замечено».

**Связка с визуалом:** каждой фразе соответствует своя марка-иллюстрация (раздел 5).

## 5. Визуальное направление

**Референсы по атмосфере:** Kinfolk, Cereal Magazine.
**Референс по лаконичности и тактильным анимациям марок:** queensjournal.app — берём **только** минимализм и механику «лепки / отлепления / замены» марок; **фото-механику НЕ берём**.

**Палитра** (ограниченная; hex — стартовая точка, финал в дизайн-ТЗ):
- молочный — фон/база
- лён — бумага/карточки
- тёплый серый — вторичный текст, тонкие линии
- приглушённый шалфейный — акцент / собранное состояние

**Пространство.** Воздух — вокруг листа (поля). Сама сетка **плотная**: 20 ячеек вплотную, разделённые тонкими линиями, как обычная аккуратная таблица — без больших промежутков. Ощущение листа/альбома, лёгкая фактура бумаги, минимум декора.

**Типографика.** Характерный editorial-серип для заголовка и фраз; тихая утилитарная гарнитура для мелких подписей. Точные шрифты — в дизайн-ТЗ.

**Марки.** Маленькие монолайн-иллюстрации с перфорированным «зубчатым» краем, в палитре, каждая тематична своему заданию. Это **signature-элемент** проекта.

**Фон.** Чёрно-белые блики солнца на воде. Тихо, низкий контраст, не отвлекает. Предпочтительно лёгкая CSS-имитация вместо тяжёлого gif.

## 6. Механика и взаимодействие

- **Состояния карточки:** «фраза» → (тап) → «марка собрана».
- **Анимация сбора:** текст мягко исчезает (fade), марка появляется с тактильным «приклеиванием» (масштаб + лёгкий поворот + мягкая тень). Ощущение — как делается марка в queensjournal.
- **Отмена («отлепить»):** да. Возможность снять марку (долгий тап или маленький контрол) и вернуть фразу — защита от случайных тапов. Со своей анимацией отлепления.
- **Счётчик прогресса:** да, но **очень тихо** — маленький, аккуратный, в уголке таблицы. Без «осталось / победи».
- **Финал:** при сборе всех 20 — плавная анимация + **«Лето замечено»**. Эмоция — открытка, которая долетела; поощрение за внимание к своей жизни. Тихо, один раз, без модального давления.
- **Экспорт-keepsake:** да. Кнопка «сохранить» → **PNG текущего состояния доски** (частично или полностью собранного), формат **сторис 9:16 (1080×1920)**. При полном сборе в композицию добавляется фраза «Лето замечено». (Память + шеринг в сторис; заодно обходит ограничение «одно устройство».)
- **Доступность (quality floor):** `prefers-reduced-motion` → упрощённые кроссфейды без вращений; видимый фокус для клавиатуры; тач-таргеты ≥ 44px.

---

## 7. Сохранение и технологии

- **Один статический сайт:** один HTML-файл (HTML + CSS + JS инлайн). Без бэкенда, без сборки.
- **Сохранение прогресса:** `localStorage` с graceful-fallback через `try/catch`. Ключ версионируем.
- **Ограничение localStorage:** одно устройство / один браузер. Прогресс не переносится телефон ↔ ноут. Принимаем; митигируем экспортом-PNG.
- **Платформы:** **десктоп — приоритетная версия**; мобильная **обязательна**, с адаптивной сеткой (на узких экранах 2 колонки). Тач-таргеты ≥ 44px.
- **Язык интерфейса:** только русский.
- **Хостинг:** **GitHub Pages** (статика). Относительные пути — учитываем возможный subpath репозитория.
- **Производительность:** лёгкий вес; оффлайн-фоллбек для шрифтов и фона.

## 8. Этапы реализации

**Этап 0 — Финализация контента.** Она пишет и утверждает 20 фраз + финальную строку. *(Готово по решениям; остаются только сами фразы за ней.)*

**Этап 1 — Дизайн (Claude Design).** Визуальный язык: палитра (финальные hex), типографика, лист/бумага, состояния (фраза / собрана / финал / экспорт-PNG), фон. → `letnee-bingo-design-tz.md`

**Этап 2 — Дизайн марок.** 20 марок-иллюстраций в утверждённом стиле (или система, генерирующая их единообразно). → `letnee-bingo-design-tz.md`

**Этап 3 — Сборка (Claude Code).** Статический сайт: сетка, тап→марка, отлепление, localStorage, счётчик, анимации, финал, PNG-экспорт, адаптив, доступность. → `letnee-bingo-code-tz.md`

**Этап 4 — Интеграция контента.** Финальные фразы и марки → в собранный сайт.

**Этап 5 — Тест + полировка + деплой.** Реальные телефоны (iOS/Android) и десктоп, `prefers-reduced-motion`, оффлайн-фоллбек; полировка анимаций; деплой на GitHub Pages.
