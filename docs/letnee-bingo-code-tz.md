# Летнее бинго — ТЗ для разработки (Claude Code)

Самодостаточный бриф на сборку статического сайта. Визуальные специфики (цвета, шрифты, SVG-марки, тайминги) берутся из дизайн-деливерабла (`letnee-bingo-design-tz.md` → результат Claude Design). Где их ещё нет — ставь аккуратные плейсхолдеры из палитры ниже и пометь `TODO: из дизайна`.

---

## 1. Что строим

Один статический сайт: лист бумаги с сеткой **4×5 = 20** ячеек. В каждой ячейке — короткая фраза-наблюдение. Тап по ячейке «собирает» наблюдение: текст исчезает, на его месте появляется маленькая **марка** (inline-SVG). Прогресс сохраняется в браузере. При сборе всех 20 — тихий финал «Лето замечено». Есть экспорт собранного альбома в PNG.

Это практика внимания **без камеры** — никаких загрузок фото, никакого бэкенда, никаких аккаунтов.

---

## 2. Технический стек и ограничения

- **Один файл `index.html`**: HTML + CSS + JS инлайн. Без фреймворков, без сборки, vanilla JS.
- **Хостинг — GitHub Pages.** Все пути относительные (учитывай, что сайт может жить на `user.github.io/repo/`). Никаких абсолютных `/`-путей к ассетам.
- **Сохранение — `localStorage`**, обёрнутое в `try/catch`: если хранилище недоступно/заблокировано, приложение всё равно работает в рамках сессии (не падает). Версионируемый ключ, напр. `letnee-bingo:v1`.
- **Платформы:** десктоп — приоритет; мобильная обязательна, адаптивная сетка (узкий экран → 2 колонки). Тач-таргеты ≥ 44px.
- **Оффлайн-фоллбек** для шрифтов (system-serif / system-sans) и фона.
- **Язык — только русский.**
- **Без внешних зависимостей по возможности.** Если для PNG-экспорта решишь взять библиотеку — только через CDN и с фоллбеком (см. §6). Предпочтителен путь без зависимостей.

---

## 3. Модель данных

Фразы и марки держим в **одном легко редактируемом массиве в начале JS** — чтобы автор курса могла менять копирайт, не трогая логику.

```js
const TASKS = [
  { id: 1,  text: "Понюхать запах свежескошенной травы", stamp: "grass" },
  { id: 2,  text: "Найти животное в облаках",             stamp: "cloud" },
  // ... всего 20
];
```

- `text` — фраза (финал придёт от автора; сейчас — заглушки из master-ТЗ).
- `stamp` — id inline-SVG марки (приходят из дизайна).
- Состояние: массив собранных id в `localStorage`, напр. `{ collected: [1, 4, 7] }`.
- Порядок ячеек = порядок в `TASKS` (4 колонки × 5 строк).

Логика и контент **развязаны**: поменять фразу = поменять строку в `TASKS`, без правок механики.

## 4. Механика

**Сбор (тап по ячейке-фразе):**
- Текст плавно исчезает (fade), марка появляется с «приклеиванием»: масштаб + лёгкий поворот + мягкая тень. Тайминги/ощущение — из дизайна.
- id добавляется в `collected`, состояние пишется в `localStorage`.

**Отлепление (отмена):**
- Снять марку и вернуть фразу. Триггер: долгий тап **или** маленький аккуратный контрол на марке (выбери менее «шумный» на десктопе и удобный на тач). Со своей анимацией отлепления.
- id убирается из `collected`, состояние сохраняется.

**Счётчик прогресса:**
- Маленький, тихий, **в уголке** доски: `N / 20`. Не акцент, без «осталось/победи».

**Восстановление:**
- При загрузке читаем `localStorage` и отрисовываем собранные марки сразу в нужных ячейках.

**Финал (collected.length === 20):**
- Плавная анимация + экран **«Лето замечено»** (тёплая открытка-поощрение, без модального давления, без салюта). Показываем один раз на переход к полному состоянию; доска остаётся доступной.

**Сброс (для теста/рестарта):**
- Тихий, неприметный контрол (напр. в футере) с подтверждением — очищает `collected`.

---

## 5. Адаптив и доступность (quality floor)

- Десктоп: лист по центру, комфортные поля, сетка 4×5.
- Мобайл: 2 колонки, та же логика, крупные тач-таргеты.
- `prefers-reduced-motion`: упрощённые кроссфейды без вращений/масштабов.
- Клавиатура: ячейки фокусируемы, видимый фокус, Enter/Space = собрать/отлепить.
- ARIA: ячейка-кнопка с понятным label («Собрать: <фраза>» / «Отлепить марку»); счётчик и финал — через `aria-live` тактично.
- Контрастность текста читаемая в рамках тихой палитры.

## 6. Экспорт в PNG (формат сторис)

Кнопка «сохранить» → генерирует **PNG** текущего состояния доски и скачивает файл.

- **Доступен в любой момент**, не только на финале. Экспортит **текущее** состояние: часть ячеек — текст-задания, часть — уже собранные марки. Снимок доски как есть.
- **Формат — Instagram Stories: 9:16, 1080×1920** (с ретина-рендером по факту, итог ресайзим/компонуем под 1080×1920).
- Когда собраны все 20 — в композицию добавляется фраза **«Лето замечено»**. До этого — тихий заголовок/подпись по макету из дизайна.
- Композиция (поля под вертикаль, расположение сетки и заголовка) — по макету `4D` из дизайн-ТЗ.
- **Предпочтительный путь (без зависимостей):** рисуем композицию на offscreen `<canvas>` размера 1080×1920 — фон-бумага, заголовок, сетка ячеек (текст для несобранных, SVG-марка для собранных). SVG-марки переносим через `Image` с `src = "data:image/svg+xml;..."` → `drawImage`; затем `canvas.toDataURL('image/png')` → скачивание через `<a download>`.
- **Фоллбек:** `html2canvas` с CDN (рендер скрытого DOM-узла, свёрстанного под 1080×1920) — если ручной canvas окажется неудобным из-за переноса текста заданий. Но сначала пробуем canvas вручную.

## 7. Структура и качество кода

- Чисто разделить: `TASKS` (данные) · рендер · стейт/persistence · анимации · экспорт.
- CSS — переменные палитры/типографики в `:root` (токены из дизайна), аккуратные селекторы без взаимного перекрытия паддингов.
- Комментарии — там, где автору/правкам важно (блок `TASKS`, ключ localStorage, точки подмены ассетов).
- Никаких `console`-шумов в проде.

## 8. Готовность (definition of done)

- [ ] `index.html` открывается локально и работает целиком (тап, отлепление, счётчик, финал, экспорт).
- [ ] Прогресс переживает перезагрузку (localStorage), и приложение не падает при заблокированном хранилище.
- [ ] Десктоп и мобайл (2 колонки) выглядят и работают корректно.
- [ ] `prefers-reduced-motion`, фокус, ARIA — на месте.
- [ ] PNG-экспорт работает в любой момент, даёт картинку 1080×1920 (сторис) с текущим состоянием доски; при полном сборе включает «Лето замечено».
- [ ] Фразы и марки меняются правкой `TASKS`, без касания логики.
- [ ] Раскладывается на GitHub Pages как есть (относительные пути).
